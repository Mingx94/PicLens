# Architecture

## Goal

ImageViewerWin is a native Windows implementation of the Electron ImageViewer product. It should preserve product behavior where it matters while using native WinUI 3 patterns instead of Electron-specific chrome or IPC.

## Solution Layout

```text
ImageViewerWin.slnx
ImageViewerWin/                  WinUI 3 app, XAML views, app assets, window setup
src/ImageViewerWin.Application/  Service contracts and deterministic operation planning
src/ImageViewerWin.Core/         Pure models and domain rules
src/ImageViewerWin.Infrastructure/ JSON settings, filesystem, image, and recycle-bin services
tests/ImageViewerWin.Application.Tests/ xUnit tests for application behavior
tests/ImageViewerWin.Core.Tests/ xUnit tests for pure domain behavior
tests/ImageViewerWin.Infrastructure.Tests/ xUnit tests for infrastructure behavior
docs/                            Project documentation
tools/                           Repo-local maintenance scripts
artifacts/portable/              Generated no-install release outputs
```

## WinUI App

The WinUI app uses the official MVVM template with CommunityToolkit.Mvvm. The root window is `MainWindow`, which hosts `MainPage` in a frame, uses Mica, sets the native app icon, and sizes the main window to the Electron parity target of `1220x820` logical pixels.

`MainPage` is the current native shell:

- System and user favorites list, including add, remove, and user reorder
- Folder tree rooted at the best matching favorite
- Library command bar for history, sort key/direction, recursive mode, and file operations
- Mixed folder/image grid with real thumbnails for still images
- Browser-style mouse side-button folder history navigation
- File-operation status bar

`MainPageViewModel` coordinates service-backed browsing, settings persistence, selection, conservative file operations, and opening the secondary viewer window. XAML code-behind is limited to WinUI-only work such as pickers, dialogs, drag/drop events, and launching windows.

`ImageViewerWindow` displays an `ImageSequenceSnapshot` with previous/next navigation, zoom in/out/reset, pointer wheel zoom, drag pan, fullscreen toggle, keyboard shortcuts, Escape close/focus behavior, and unsupported animated-image feedback.

## Assets

The Windows 11 style app icon source is generated by `tools/generate_app_icon.py`. It writes the multi-size `Assets\AppIcon.ico` used by the native window plus the WinUI logo PNGs used by the app manifest.

## Core Domain

`ImageViewerWin.Core` owns product rules that should be testable without WinUI:

- Supported image extension detection
- Animated GIF and WebP detection
- List sorting with folder-first behavior
- Settings defaults and patch merge
- Startup folder fallback
- Immutable image sequence snapshot creation
- Zoom clamping and pointer-anchored wheel zoom

Keep filesystem, Windows UI, thumbnail codecs, and recycle-bin behavior out of Core.

## Application And Infrastructure

`ImageViewerWin.Application` owns:

- Service contracts for settings, favorites, scanning, image data, and file operations
- Deterministic rename planning and validation

`ImageViewerWin.Infrastructure` owns:

- JSON settings persistence
- System and user favorite folder loading
- Direct and recursive folder scanning with canonical directory de-duplication
- Image file loading helpers
- JPG conversion that preserves originals and skips collisions
- Recycle-bin trash operations
- Drop-target batch rename execution that advances past existing sequence targets

This split keeps WinUI views thin while preserving Electron parity rules in reusable, tested code.
