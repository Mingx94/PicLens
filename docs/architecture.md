# Architecture

## Goal

ImageViewerWin is a native Windows implementation of the Electron ImageViewer product. It should preserve product behavior where it matters while using native WinUI 3 patterns instead of Electron-specific chrome or IPC.

## Solution Layout

```text
ImageViewerWin.slnx
ImageViewerWin/                  WinUI 3 app, XAML views, app assets, window setup
src/ImageViewerWin.Core/         Pure models and domain rules
tests/ImageViewerWin.Core.Tests/ xUnit tests for pure domain behavior
docs/                            Project documentation
tools/                           Repo-local maintenance scripts
artifacts/portable/              Generated no-install release outputs
```

## WinUI App

The WinUI app uses the official MVVM template with CommunityToolkit.Mvvm. The root window is `MainWindow`, which hosts `MainPage` in a frame, uses Mica, sets the native app icon, and sizes the main window to the Electron parity target of `1220x820` logical pixels.

`MainPage` is the current native shell:

- Favorites list
- Folder tree
- Library command bar
- Mixed folder/image grid
- File-operation status bar

The current UI data is seeded by `MainPageViewModel`. Future lanes should replace seeded data with services without moving business rules into XAML code-behind.

## Assets

The Windows 11 style app icon source is generated by `tools/generate_app_icon.py`. It writes the multi-size `Assets\AppIcon.ico` used by the native window plus the WinUI logo PNGs used by the app manifest.

## Core Domain

`ImageViewerWin.Core` owns product rules that should be testable without WinUI:

- Supported image extension detection
- Animated GIF and WebP detection
- List sorting with folder-first behavior
- Settings defaults and patch merge
- Startup folder fallback
- Immutable image sequence snapshot creation
- Zoom clamping and pointer-anchored wheel zoom

Keep filesystem, Windows UI, thumbnail codecs, and recycle-bin behavior out of Core.

## Future Application And Infrastructure Lanes

Planned next projects:

- `ImageViewerWin.Application`: view models and service interfaces for browsing, selection, dialogs, and viewer windows.
- `ImageViewerWin.Infrastructure`: settings JSON, system favorites, folder scanning, thumbnails, image loading, recycle-bin trash, and file-management operations.

This split keeps WinUI views thin while preserving Electron parity rules in reusable, tested code.
