# 產品規格

## 文件目的

這份文件定義 PicLens 的產品行為、使用者流程與品質要求。它刻意不包含使用框架、技術選型、專案分層、測試工具、build 指令或發佈指令；那些內容由其他工程文件維護。

本文件只描述目前已實作或已由現有文件明確定義的產品範圍。未被明確定義的方向列在「待確認」中，不在此直接推定。

## 產品定位

PicLens 是 Windows / Linux 桌面圖片整理與檢視工具，聚焦在本機資料夾中的大量圖片瀏覽、選取、批次整理與快速檢視。

產品優先順序：

- 讓使用者以資料夾為中心快速進入圖片集合。
- 清楚呈現目前資料夾、項目數、排序狀態、選取狀態與檔案操作結果。
- 對檔案操作採保守策略，避免覆寫、避免刪除原檔、失敗時持續處理其他項目。
- 讓批次操作在執行前後都有可理解的狀態回饋。
- 在後續排查時保留足夠的錯誤紀錄。

## 目標使用情境

- 使用者需要整理本機資料夾中的大量圖片。
- 使用者需要在同一個工作區內瀏覽資料夾、查看縮圖、排序、選取圖片並執行批次操作。
- 使用者需要快速開啟單張圖片，以主視窗內嵌檢視器進行上一張/下一張、縮放、平移與全螢幕檢視。
- 使用者需要在不破壞原始檔案的前提下轉換 JPG、重新命名或將圖片移至回收筒。

## 產品範圍

PicLens 處理使用者本機檔案系統中的資料夾與圖片，不定義雲端同步、帳號系統、遠端圖庫或跨裝置同步。

執行時介面文字以繁體中文（台灣）為目前產品語系。

## 支援內容

支援的圖片副檔名：

- `jpg`
- `jpeg`
- `png`
- `bmp`
- `webp`
- `gif`

動畫 GIF 與動畫 WebP 會被辨識為動畫圖片，但目前不支援播放；檢視時必須顯示清楚的無法預覽提示。

未經規格更新前，本產品不定義 AVIF、HEIC、TIFF、SVG 或動畫播放支援。

## 啟動與資料夾選取

- App 啟動時，如果上一個開啟資料夾仍可使用，應還原該資料夾。
- 如果沒有可用的上一個資料夾，使用者需明確選擇資料夾後才進入圖庫。
- 未選擇資料夾時，主畫面應顯示可執行的選擇資料夾入口。
- 使用者可在主畫面重新選擇資料夾。
- 選擇新資料夾後，圖庫內容、資料夾樹、狀態文字與選取狀態需同步更新。

## 主畫面圖庫

主畫面需提供以下能力：

- 顯示目前資料夾路徑與目前資料夾名稱。
- 顯示以目前資料夾為根的資料夾樹。
- 顯示資料夾與支援圖片檔案的格狀圖庫。
- 顯示目前圖庫項目數。
- 只針對可見或已顯示的靜態圖片載入縮圖。
- 在縮圖載入失敗時保留可操作的圖庫狀態，並寫入錯誤紀錄。
- 使用者可調整縮圖大小，且設定需保存。

## 導覽

- 使用者可透過上一頁與下一頁在資料夾歷史中移動。
- 滑鼠側鍵也應使用同一組資料夾歷史。
- 使用者可重新整理目前圖庫。
- 使用者點擊資料夾項目時，應進入該資料夾。
- 導覽、重新整理、排序與切換包含子資料夾時，不可留下舊資料夾的選取狀態。

## 排序與顯示範圍

排序選項：

- 名稱由小到大
- 名稱由大到小
- 修改時間最舊到最新
- 修改時間最新到最舊

顯示範圍：

- 僅目前資料夾
- 含子資料夾

包含子資料夾設定需保存。切換設定後，圖庫需重新載入並清除目前選取。

只瀏覽目前資料夾時，資料夾項目應優先顯示在圖片前方，並採接近 Windows 檔案總管的自然排序。

## 選取與情境操作

- 使用者可在圖庫中選取圖片。
- 左鍵點選圖片只更新選取狀態，不顯示浮動操作列。
- 右鍵點選圖片時，應顯示圖片情境選單。
- 右鍵點在已選取圖片上時，情境選單作用於目前選取；右鍵點在未選取圖片上時，需先改成只選取該圖片。
- 右鍵資料夾項目時，不提供圖片重新命名或移至回收筒操作。
- 清除選取需同時清除畫面選取狀態與內部選取狀態。
- 重新載入圖庫、切換資料夾、切換包含子資料夾、排序或完成檔案操作後，不可留下過期選取。

圖片情境選單需提供：

- 重新命名：僅適用單張選取圖片。
- 移至回收筒：可適用一張或多張選取圖片。

## 檔案操作

檔案操作需遵守保守規則：

- 移除類操作應移至作業系統回收筒。
- 轉換 JPG 時保留原始檔案。
- 遇到目標檔名衝突時跳過，不覆寫。
- 批次操作需回報總數、成功數、失敗數與逐項結果。
- 批次操作中單一項目失敗時，其他項目仍應繼續處理。
- 失敗項目需寫入錯誤紀錄，包含足夠的來源、目標與原因資訊。

主畫面需提供：

- 將目前顯示項目轉為 JPG。
- 清除同名非 JPG 檔案。
- 將選取圖片轉成 JPG。
- 將單張選取圖片重新命名。
- 將一張或多張選取圖片移至回收筒。

重新命名規則：

- 單張重新命名只允許修改不含副檔名的檔名。
- 同名重新命名應視為略過。
- 目標已存在時，不可覆寫。

## 拖放重新命名

- 使用者可在 app 內將一張或多張圖片拖到另一張圖片上，觸發批次重新命名流程。
- 拖曳中需顯示可理解的拖曳預覽。
- 可放下的目標圖片需有明確提示。
- 執行前需顯示重新命名預覽並取得使用者確認。
- 使用者取消確認時，不應修改任何檔案。
- 重新命名序號需以目標圖片不含副檔名的檔名為基準，找出最小可用序號。
- 判斷目標占用時，需以不含副檔名的檔名為準，避免不同副檔名覆蓋同一序號語意。

## 圖片檢視器

使用者開啟圖片後，應進入主視窗內嵌圖片檢視器。

檢視器需提供：

- 主視窗 app bar 顯示目前圖片名稱，檢視器內只保留導覽與縮放控制列及主要圖片畫布。
- 上一張與下一張。
- 滑鼠滾輪縮放。
- 放大、縮小與重設縮放。
- 拖曳平移。
- 鍵盤導覽。
- 全螢幕切換。
- 顯示目前圖片名稱。
- 對不支援預覽的動畫圖片顯示清楚提示。

檢視器的圖片序列在開啟時固定；主畫面後續重新載入不應直接改動已開啟檢視器的導覽序列。

Escape 行為：

- 全螢幕時，Escape 離開全螢幕。
- 非全螢幕時，Escape 關閉檢視器並回到主畫面焦點。

## 狀態與錯誤回饋

- 主要操作需在畫面上提供狀態回饋。
- 警告或失敗狀態需可被使用者理解。
- 執行時錯誤需寫入 app log，方便後續排查。
- 錯誤紀錄至少需涵蓋啟動、主畫面導覽、縮圖載入、資料夾樹載入、檔案操作、拖放重新命名逐項失敗，以及圖片檢視視窗生命週期。

目前 app log 位置：

```text
Local app data 下的 `PicLens/Logs/PicLens.log`
```

## 品質要求

- 本機資料夾與大量圖片瀏覽時，縮圖載入不可阻塞整個圖庫操作。
- 有問題的圖片或圖片解碼流程不可永久卡住後續縮圖載入。
- 使用者快速捲動時，不應持續處理已不在畫面上的縮圖請求。
- 檔案操作不可靜默失敗。
- 成功、跳過與失敗結果需能反映到使用者可見狀態。
- 產品行為需避免使用者誤以為已刪除、覆寫或改名成功。

## 目前不定義

以下項目目前不由本文件定義；若要加入，需要另行確認產品意圖並更新規格：

- 雲端同步或遠端圖庫。
- 帳號、登入或多使用者權限。
- 圖片編輯、裁切、濾鏡或標註。
- 評分、標籤、相簿、收藏或中繼資料編輯。
- 重複圖片偵測。
- 動畫播放。
- AVIF、HEIC、TIFF、SVG 等額外格式。
- 跨平台、行動版或網頁版。
- 正式效能門檻，例如最大圖片數、最大資料夾深度或縮圖載入時間目標。
