# Writing Style

Source: https://learn.microsoft.com/zh-tw/windows/apps/design/style/writing-style

Source snapshot: Microsoft Learn zh-tw page reviewed on 2026-06-06.

UI writing affects usability as much as layout and controls.

## Guidelines

- Be warm, helpful, and concise.
- Do not blame the user, especially in errors.
- Use plain language that the audience understands.
- Put the main point first.
- Use active voice and action-oriented wording.
- Keep paragraphs short; users scan UI text.
- Refer to the user directly. In Traditional Chinese, use natural direct wording and avoid stiff machine-translated phrasing.
- Explain uncommon abbreviations the first time; avoid abbreviations when localizing if they may not carry across languages.
- Use periods/full stops for complete sentences in tooltips, errors, and dialogs; avoid punctuation on short labels, checkbox text, radio labels, and buttons unless locale convention requires it.

## Errors

Error messages should:

- Explain what happened.
- Explain what happens next when relevant.
- Offer a practical recovery action when possible.
- Exclude diagnostic detail unless it helps the user recover.
- Avoid blaming the user.

## Dialogs And Buttons

- Dialog text should be brief because dialogs interrupt flow.
- Dialog buttons should clearly answer the title/question and use consistent formatting.
- Button labels should be short action verbs or verb phrases.
