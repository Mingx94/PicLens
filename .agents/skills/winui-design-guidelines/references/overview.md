## Overall Principles

- Make the UI intuitive, polished, and consistent with Fluent Design.
- Let design foundations drive decisions early: color, type, motion, layout, materials, navigation, writing, and accessibility.
- Build from Windows conventions before introducing custom behavior.
- Prefer standard WinUI controls because they inherit theme, accessibility, input, focus, and animation behavior.
- Use the WinUI 3 Gallery when you need to compare the design to interactive examples.
- Treat the guidance as a set of foundations rather than visual decoration. Each decision should support task clarity, platform consistency, or accessibility.
